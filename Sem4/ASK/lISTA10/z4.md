w a mamy definicje maina a w b mamy niezainicjowaną zmienną globalną char main
dla linkera to po prostu etykiety funkcja main to bedzie symbol silny a zmienna słaby, wiec linker wybierze symbol silny
wiec funkcja p2 traktuje początek kodu maszynowego funkcji main jako jednobajtowa zmienna typu char, wypisuje pierwszy 
bajt kodu maszynowego maina jako liczba szesnastkowa

Symbole silne to funkcje i zainicjowane zmienne globalne czyli takie które mają przypisaną wartosc
a słabe to takie niezainicjowane

nie mogą istniec dwa silne symbole o tej samej nazwie, -> blad multiple definition
jesli jest słaby i silny to wybierze silny
jesli dwa słabe wybierze losowy

jesli w p2 zapiszemy wartosc do main, to bedzie segfault bo kod funkcji trafia do .text która jest read only, próba zapisania 
czekogolwiek jest blokowana przez system operacyjny
Jesli zainicjujemy to bedziemy mieli multiple definition
